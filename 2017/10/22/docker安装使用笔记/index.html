docker安装使用笔记

Docker是一种开源容器引擎，将应用打包安装在称之为容器的环境中，采用沙箱机制，容器之间不会有任何接口，开销相对较小，独立于真实工作环境，类似虚拟机。

Docker在kali下的安装与使用，刚开始着实失败了几次，首先使用apt安装，之后按照菜鸟教程里的安装方式，因为感觉ununtu和kali相近，安装应该可以，失败，接着使用了官网里debian的安装流程，同样失败，这时就有些不解了，kali和debian是同系列，也可以说kali是基于debian,怎么不可以呢，最后才知道自己想当然了一次，kali基于debian wheezy,这次安装成功。下面是安装步骤。

  1. 对于Debian wheezy用户i必须提供反向端口，启用backports
    (1)用任意编辑器打开/etc/apt/sources.list.d/backports.list文件，如果没有这个文件，就创建一个。
    (2)删除现有所有条目后添加一条新的反向端口条目

    1. 更新软件包信息

  2. 更新apt存储库
    (1)如果之前安装过docker，则执行下列命令，清楚旧的存储库
    （2)更新存储库信息，确保apt使用https方法，并安装CA证书
    (3)添加新的GPG key（这里可能会失败几次，提示从公钥服务器接受失败，多试几次就可以了）
    (4)用任意编辑器打开/etc/apt/sources.list.d/docker.list文件，如果不存在，就创建一个
    删除现有的所有条目，并添加新的条目:deb https://apt.dockerproject.org/repo debian-wheezy main
    (5)更新apt存储库，并验证apt存储库是正确的

  3. 安装Docker
    　　1、更新apt存储库
    　　2、安装Docker
    　　3、启动Docker服务
    　　4、更新Docker

  4. 卸载Docker
    　　(1)卸载docker软件包
    　　(2)卸载docker所有程序包和依赖关系

  5. 如果要删除所有图像，容器和卷，请运行以下命令

Docker基本使用命令

```
docker run [option] mirror command #运行容器
-d:让容器在后台运行。
-P:将容器内部使用的网络端口映射到我们使用的主机上。
-t:在新容器内指定一个伪终端或终端。
-i:允许你对容器内的标准输入 (STDIN) 进行交互。

docker stop container id / name #停止容器
docker ps #查看运行容器
docker logs name #查看容器内输出
docker rm determined_swanson #移除web容器信息
docker images #列出镜像列表
docker pull mirrorname #拉取镜像
docker search mirrorname #查找镜像

更新镜像，更新镜像之前，我们需要使用镜像来创建一个容器。
docker run -t -i ubuntu:15.10 /bin/bash
在运行的容器内使用 apt-get update 命令进行更新。
在完成操作之后，输入 exit命令来退出这个容器。
此时容器是按我们的需求更改的容器。我们可以通过命令 docker commit来提交容器副本。
runoob@runoob:~$ docker commit -m=”has update” -a=”runoob” e218edb10161 runoob/ubuntu:v2
各个参数说明：

-m:提交的描述信息

-a:指定镜像作者

e218edb10161：容器ID

runoob/ubuntu:v2:指定要创建的目标镜像名

构建镜像

我们使用命令 docker build ， 从零开始来创建一个新的镜像。为此，我们需要创建一个 Dockerfile 文件，其中包含一组指令来告诉 Docker 如何构建我们的镜像。

runoob@runoob:~$ cat Dockerfile
FROM centos:6.7
MAINTAINER Fisher “fisher@sudops.com”

RUN /bin/echo ‘root:123456’ |chpasswd
RUN useradd runoob
RUN /bin/echo ‘runoob:123456’ |chpasswd
RUN /bin/echo -e “LANG=\”en_US.UTF-8\”” >/etc/default/local
EXPOSE 22
EXPOSE 80
CMD /usr/sbin/sshd -D

每一个指令都会在镜像上创建一个新的层，每一个指令的前缀都必须是大写的。

第一条FROM，指定使用哪个镜像源

RUN 指令告诉docker 在镜像内执行命令，安装了什么。。。

然后，我们使用 Dockerfile 文件，通过 docker build 命令来构建一个镜像。

runoob@runoob:~$ docker build -t runoob/centos:6.7 .

参数说明：

-t ：指定要创建的目标镜像名

. ：Dockerfile 文件所在目录，可以指定Dockerfile 的绝对路径

docker tag #创建镜像标签
