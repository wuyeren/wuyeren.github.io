几种数据库的介绍和比对

本文主要涉及以下几种数据库：mysql， sqlite， redis， mongodb， memcached. 其中mysql和sqlite是关系型数据库，即是建立在关系模型上的数据库，关系模型包括关系数据结构，关系操作集合，关系完整性约束三个部分，简单点来说是维护二维行列表格的数据库，redis，mongodb和memcached是非关系型数据库，主要以键值对的形式进行数据的存储和查询，此外，Redis支持五种数据类型：string（字符串），hash（哈希），list（列表），set（集合）及zset(sorted set：有序集合);memcached是内存对象缓存系统，提供缓存机制的功能，且存储在内存中，mongodb和memcached是分布式系统。

数据库也是程序，因此会涉及到安装，配置，使用，本文介绍数据库在windows和linux系统下的安装，先说下两种系统安装软件的方式，windows下安装软件相对来说比较简单，因为软件都有相应的图形安装方式，基本一路默认即可，这里重点说一下linux安装软件的方式，第一种是源码安装，也就是下载程序源代码，解压随后在本地编译安装，具体的流程包括三步：
./configure [检查编译情况，主要是查看系统，依赖等]
make [安装]
make install [安装]
make clean [清理]
make uninstall [卸载]
第二种是通过包管理器进行安装，比如rpm, dpkg 等包管理器安装，使用help选项可以查看命令语法，第三种是通过apt等命令在线安装，同样通过help选项可以查看命令语法。同样的，这几种方式也可以用于卸载，只是相应的命令选项不同。

下面说一下几种数据库的安装，主要就是上面几种安装方式：
mysql
windows:
下载window版本的mysql安装包，并解压安装包，双击 setup.exe 文件，接下来你只需要安装默认的配置点击”next”即可，默认情况下安装信息会在C:\mysql目录中。接下来你可以通过”开始” ，在搜索框中输入 “ cmd” 命令 ，在命令提示符上切换到 C:\mysql\bin 目录，并输入一下命令：
mysqld.exe –console
如果安装成功以上命令将输出一些mysql启动及InnoDB信息。
linux:
yum install mysql
yum install mysql-server
yum install mysql-devel
mysqladmin –version [验证是否安装]
redis：
windows:
下载windows版本的安装包，解压安装包，打开cmd通过cd命令进入相应的安装包目录，执行redis-server.exe redis.windows.conf
linux:
wget http://download.redis.io/releases/redis-2.8.17.tar.gz
tar xzf redis-2.8.17.tar.gz
cd redis-2.8.17
make
cd src
./redis-server redis.conf
[ubuntu]:
sudo apt-get update
sudo apt-get install redis-server
启动 Redis
redis-server
sqlite
widows:
下载 sqlite-tools-win32-.zip 和 sqlite-dll-win32-.zip 压缩文件。
创建文件夹 C:\sqlite，并在此文件夹下解压上面两个压缩文件，将得到 sqlite3.def、sqlite3.dll 和 sqlite3.exe 文件。
添加 C:\sqlite 到 PATH 环境变量，最后在命令提示符下，使用 sqlite3 命令，将显示如下结果
C:>sqlite3
SQLite version 3.7.15.2 2013-01-09 11:53:05
Enter “.help” for instructions
Enter SQL statements terminated with a “;”
sqlite>

linux:
系统自带sqlite，如果没有运行以下命令：
tar xvfz sqlite-autoconf-3071502.tar.gz
cd sqlite-autoconf-3071502
./configure –prefix=/usr/local
make
make install

mongodb
windows:
下载二进制安装包，解压后运行程序进行图形界面安装，之后在盘符根目录下创建/data/db目录。
linux:
curl -O https://fastdl.mongodb.org/linux/mongodb-linux-x86_64-3.0.6.tgz
tar -zxvf mongodb-linux-x86_64-3.0.6.tgz
mv mongodb-linux-x86_64-3.0.6/ /usr/local/mongodb
export PATH=/bin:$PATH
mkdir -p /data/db
安装之后的一些配置

创建目录，执行下面的语句来创建数据库和日志文件的目录
mkdir c:\data\db
mkdir c:\data\log
创建配置文件
创建一个配置文件。该文件必须设置 systemLog.path 参数，包括一些附加的配置选项更好。
例如，创建一个配置文件位于 C:\mongodb\mongod.cfg，其中指定 systemLog.path 和 storage.dbPath。具体配置内容如下：
systemLog:
destination: file
path: c:\data\log\mongod.log
storage:
dbPath: c:\data\db
安装 MongoDB服务
通过执行mongod.exe，使用–install选项来安装服务，使用–config选项来指定之前创建的配置文件。
C:\mongodb\bin\mongod.exe –config “C:\mongodb\mongod.cfg” –install

memcached
windows:
解压下载的安装包到指定目录。
使用管理员身份执行以下命令将 memcached 添加来任务计划表中：
schtasks /create /sc onstart /tn memcached /tr “‘c:\memcached\memcached.exe’ -m 512”
linux:
Linux系统安装memcached，首先要先安装libevent库。
sudo apt-get install libevent libevent-deve
（Ubuntu/Debian）
yum install libevent libevent-deve （Redhat/Fedora/Centos）
Ubuntu/Debian
sudo apt-get install memcached
Redhat/Fedora/Centos
yum install memcached
FreeBSD
portmaster databases/memcached
以上就是简单的一些安装方式，至于运行方面一般都是先启动服务，接着使用客户端程序进行数据库的操作，下面介绍一下几种数据库的运行和连接方式。
mysql
首先启动服务，windows平台打开cmd使用net start mysqld命令，或者使用任务管理器、计算机管理等图形接口开启服务，linux平台使用service start mysql开启服务，接着使用mysql -uroot -p以管理员权限执行操作。
sqlite
直接在命令终端键入sqlite3进行操作。
redis
windows打开cmd通过cd命令进入安装目录，接着使用redis-server.exe启动服务，打开另一个窗口，使用redis-cli.exe -h localhost -p 6379连接数据库进行操作，linux中相应的命令为redis-server; Redis-cli
memcached
这个数据库比较简单，直接使用以下命令连接：
telnet host port
mongodb
首先启动服务
widows：C:\mongodb\bin\mongod --dbpath c:\data\db
linux：service mongod start
连接数据库
windows：C:\mongodb\bin\mongo
linux：cd [mongodir]; ./mongo
现在总结下运行方式，不仅仅包括以上数据库，也包括其它服务软件，首先开启服务，开启服务可以直接运行各软件bin目录下的二进制文件，或者使用net命令和service命令或者图形界面间接运行二进制文件，服务运行之后同样可以运用这些方式停止服务，只是使用不同的选项参数，服务运行之后，通过bin目录下的客户端二进制文件进行连接，或者通过一些第三方软件进行连接，这样比较容易操作。
